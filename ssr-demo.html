🥯 SSR Demo - Simple Greeting Component with Bun

📋 Setup Complete!

Your Lit component is now ready for server-side rendering (SSR) using Bun for ultra-fast performance. This demo shows you how to use the SSR implementation.

🛠️ How to Run SSR

Follow these steps to see your component rendered server-side:

1. Build your components:

bun run build

2. Start the SSR server:

bun run ssr:serve

3. Open your browser:

http://localhost:3000

4. Try with different names:

http://localhost:3000/?name=Alice

📦 What SSR Provides

🥯 Bun Performance

Powered by Bun for ultra-fast TypeScript execution, built-in bundling, and optimized HTTP server.

⚡ Faster Initial Load

Components render immediately without waiting for JavaScript to load and execute.

🤖 SEO Friendly

Search engines and web crawlers can read your component content directly from the HTML.

💪 Progressive Enhancement

Your page works even if JavaScript fails to load or is disabled.

🔄 Hydration Support

Components become interactive after JavaScript loads, maintaining their server-rendered state.

📁 Files Created for SSR

  • ssr-server-bun.ts - Bun-optimized SSR server with built-in TypeScript support
  • src/ssr-server.ts - Node.js compatible SSR server (fallback)
  • src/client-hydrate.ts - Client-side hydration utilities
  • package.json - Updated with Bun SSR scripts and types
  • tsconfig.json - Updated to include Node.js and Bun types

🔧 Bun Server Features

The Bun SSR server includes:

  • ✅ Native TypeScript support (no build step needed)
  • ✅ Ultra-fast startup and response times
  • ✅ Server-side rendering of your simple-greeting component
  • ✅ Built-in HTTP server with modern fetch API
  • ✅ Declarative Shadow DOM support
  • ✅ Client-side hydration with polyfills
  • ✅ Query parameter support (?name=YourName)
  • ✅ Efficient file serving with Bun.file()
  • ✅ Health check endpoint (/health)
  • ✅ Error handling and 404 pages
  • ✅ Hot reload support with --watch flag

🧪 Testing SSR

To verify SSR is working:

  1. Disable JavaScript in your browser dev tools
  2. Reload the page at http://localhost:3000
  3. You should still see the styled component content
  4. View source to see the rendered HTML includes your component's shadow DOM
